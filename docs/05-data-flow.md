# 데이터 흐름

## 목적

이 문서는 실제 DB 스키마를 확정하기 위한 문서가 아니다.

다정 MVP에서 어떤 페이지와 기능이 어떤 데이터를 필요로 하는지 정리하기 위한 문서이다.

## 기본 흐름

사용자 요청
→ 다정 AI
→ MCP Server
→ Backend API
→ DB 또는 Dummy Data
→ 응답 생성
→ 카드 UI 또는 화면 표시

## 주요 데이터 후보

- 기업 정보
- 메뉴 정보
- 메뉴 옵션
- 사용자 정보
- 주문 정보
- 주문 상세 항목
- 결제 정보
- 포인트 정보
- 키오스크 세션 정보
- AI 카드 응답 정보

## 사용하는 화면

### 키오스크 페이지

- 기업 정보
- 메뉴 목록
- 메뉴 옵션
- 장바구니
- 주문 생성 결과

### 관리자 페이지

- 주문번호
- 대기번호
- 주문시간
- 주문 상태
- 결제 상태
- 포인트 적립 여부

### 다정 AI 채팅

- 사용자 요청
- 메뉴 추천
- 주문 요약
- 결제 확인
- 포인트 안내

# Codex 작업 원칙

## 기본 순서

1. 현재 repo 구조를 읽습니다.
2. `README.md`, `todo.md`, `docs/mvp_process.md`, `docs/02_Phase_Plan.md`를 확인합니다.
3. 사용자가 요청한 Phase의 범위를 확인합니다.
4. 해당 Phase에 필요한 파일만 생성하거나 수정합니다.
5. 검증 명령을 실행하고 결과를 보고합니다.

## 금지 사항

- 전체 프로젝트를 한 번에 새로 갈아엎기
- 현재 Phase 밖 기능을 임의로 구현하기
- 실제 카드번호, 결제 비밀값, 민감정보 저장하기
- 실제 Toss 실결제 처리하기
- 실제 코레일 예약/발권 자동화 구현하기
- 비공식 자동화 또는 스크래핑으로 코레일 예약 처리하기
- B/C Mock 키오스크를 실제 주문 DB와 연결하기
- 관리자 페이지 없이 주문 API만 구현하고 끝내기

## 보고 형식

작업 완료 시 아래 내용을 포함합니다.

```text
1. 생성/수정한 파일 목록
2. 핵심 변경 내용
3. 실행 또는 검증 명령과 결과
4. 다음 Phase에서 이어갈 일
```

## Commit 규칙

Commit message는 사용자 지정 convention을 따릅니다.

```text
[종류] 파일명 실제내용
```

예시:

```text
[docs] MVP 문서 실행 흐름 작성
[feat] backend/app 주문 API 추가
[refactor] 프로젝트전체 React19로 마이그레이션
```
